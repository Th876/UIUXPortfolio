Challenge

Cake Wrecks is the go-to blog for fans seeking a good laugh at baking mishaps. However, its outdated design and tricky navigation make it difficult for fans to fully enjoy the content.

So, the question was:
How can we make Cake Wrecks' interface not just easier, but also more fun to navigate?

Our Solution

We started by collaboratively identifying key usability issues and gathering user feedback. From these insights, I took on the task of redesigning the website from the ground up.

The goal? A modern, sleek interface with enhanced navigation that allows users to effortlessly explore, share, send, and inquire about cake fails across all devices.

Refining the Design

To tackle the mixed reviews, I went back to the drawing board. This iterative design process led me to design a more organized list that makes finding bakeries both intuitive and efficient. From there, I began the A/B testing.

I brought in six new participants from the screener survey pool who had an interest in cake fails. They performed the task of finding a bakery in Atlanta, Georgia, and shared their thoughts on the design.

Interestingly, the test results showed that users were equally split between the interactive map and the list, which led to my final redesign decision‚Äîa toggle feature. This feature allows users to toggle between the interactive map view (now an embedded map) and a list view.

Here's how each option of the toggle feature serves different user needs:

*Overlay extends to full screen in the actual prototype
  • Interactive Map View: A modern and engaging way for users to discover bakeries, enhancing the browsing experience. This is the default view to address user frustration with outdated features.
  • List View: A structured and straightforward method to find specific bakeries. It is an alternative for users who prefer a more traditional, organized approach to navigation.

Takeaways

Impact

The Cake Wrecks redesign has transformed the way users interact with the site. After implementing the new features and putting them to the test, users felt the site was not only easier to navigate but also way more engaging.¬†

The final design not only enhances the user experience by making the site more inclusive and efficient, but it also sets the stage for future growth and continued user satisfaction‚Äîit‚Äôs a win-win for everyone!

What I learned

This project has been a valuable learning experience. Here are some key things I learned throughout the process:

  • The Power of Iterative Design:By running multiple observation and A/B tests, I saw firsthand how refining designs based on user feedback boosts usability and effectiveness.
  • Interpreting A/B Test Results:A/B testing sometimes reveals complex user preferences. Combining elements from different versions may often be more effective than choosing just one.
  • Importance of Accessibility: Accessibility goes beyond compliance. Adding a floating accessibility icon with a hide option demonstrates the need for equity-focused design since all users should have a fair and equitable experience.
  • Designing for Simplicity:This redesign reinforced that simplicity in design is key. Finding the balance between functionality and ease of use makes for a more enjoyable user experience.
